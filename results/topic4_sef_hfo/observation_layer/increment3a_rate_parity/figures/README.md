# Increment-3a rate parity 图说明

虚拟 SEEG 观测层在 LIF rate field 上的四对照验收图。估计子 = `endpoint_centroid_axis`，
锁定阈值 = 方向误差 < 25°、参与触点 ≥ 7、各向同性可读性 < 0.3。检测前端口径见 spec §4.2
（事件窗 = 源场活跃比例、participation margin = 10%，rate-substrate 锁 + sensitivity 已证不造结论）。

### rate_parity_overview.png

四个 panel 各对应一个对照，每个对照隔离一个混淆源，柱高 = 方向误差（°）、虚线 = 25° 门、柱上标 n=参与触点数。
绿 = 过、灰 = 触点不足（insufficient）、红 = 真失败。
C1 连接轴跟踪（θ_EE=0/45/90°，电极固定）三档全绿（误差 1.1/3.3/1.1°）；
C2 踢点位置（θ_EE 固定 45°、踢点垂直偏移 ±0.35 半片）三档全绿（20.5/3.3/20.5°，对称）；
C3 电极杆整体旋转（0/30/60/90°）四档全绿（最大 7.3°）；
C4 各向同性（AR=1 中心踢）右栏画 direction_readability，过 = 采够触点（n_part=12）但读不出稳定方向。

**关注点**：四栏是否全绿、所有柱是否在 25° 虚线以下、C4 是否「采够但无方向」（不是触点不足蒙混）。

### rate_readout_headline_theta45.png

主条件（C1 θ_EE=45°）的单事件读出链诊断（spec §13 锁定的「每张声称 SEEG 读出的图都要画全链条」）。
A 几何：24mm 片、4mm 触点、3 根非平行电极杆、踢点（红星，在 −45° 端）、事件足迹（沿 45° 拉长的灰带）、
绿箭头 = 真实连接方向 θ_EE、橙虚线 = 从虚拟电极恢复的方向轴（两者基本重合 = 读对了）、蓝 = 参与触点、灰 = 不参与。
B 每触点活动轨迹（firing-rate-density 包络，**明确标注 NOT LFP** —— 快版读出，正式 current-LFP 见下方边界说明），
按触点错开、事件窗阴影；从 A0 到 C5 单调先后扫过 = 可见的方向顺序。
C 导出链：每触点首次越阈 lag vs 沿恢复轴投影位置，单调线性 = 模型事件 → 电极观测 → 方向估计这条链成立。

**关注点**：橙（恢复轴）是否压住绿（真方向）；B 是否单调扫过而非杂乱；C 是否单调线性而非散点；标题是否标了「NOT LFP」。

---

**LFP 结论边界（不要越界，user 2026-06-07）**：这些图是 **rate field 的 firing-rate-density 包络**读出，证明的是
「rate 主观测路在挑选过的 4mm 三杆几何下能读回方向」。**不能**据此说「真实 4mm 尺度 SNN 虚拟电极可用」——
SNN cm 尺度复核仍点不着（受阻于 SNN 点火，不是 LFP forward 模型）；current-type LFP forward 只在 2mm 小尺度
真实点着的 SNN 事件上验证过。SNN-cm 是后调，不阻塞 rate 主路。
