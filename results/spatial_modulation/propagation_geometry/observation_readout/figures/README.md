# 传播触点平面读出 — 图说明

这套图把每个传播模板(真实病人 / 模型各一)压成同一张标准化二维触点图,用来比较"模型生成的传播"和"真实病人的传播"在先后顺序与空间铺开上像不像。**所有图只表达传播顺序和采样支撑,不含原始电活动。**

## 每张图的四个子图(统一布局)

每张图固定 2×2 四个子图,回答四个独立问题:

1. **左上 — 触点散点**:每个参与触点画一个点。颜色 = 典型先后顺序(紫=早,黄=晚);点大小 = 该触点参与事件的频率;黑圈 = 这个触点落在临床标注的发作起始区内(**仅叠加展示,不进任何指标**);红色空心三角 = 投影落到比较网格外、被裁到边界的触点(下方红字标了个数)。左上角标了物理尺度换算 `1 norm = X mm`(沿轴单位长度对应多少毫米)和 `core dist = Y mm`(早端核到晚端核的中心距离),右下有 20 mm 标尺。
2. **右上 — 平滑顺序场**:把散点的先后顺序按采样支撑加权摊成连续场。早→晚的梯度在这里直接可见;没有触点采样的地方留空(灰/透明),不补也不解读。
3. **左下 — 支撑场**:哪里有采样支撑(亮 = 有触点密集采样)。它决定上面的顺序场在哪里可信。
4. **右下 — 不确定度场**:早端那团先后顺序稳不稳。亮 = 同一触点在不同事件里先后波动大。早端通常是"一小群触点轮流当早端",所以早端不确定度偏高是预期的。

**标题栏**带四个标签:`1D/2D`(采样是否退化成一条线)、`rho_x_rank`(先后顺序沿传播轴的单调性,越接近 1 越是干净的早→晚轴)、`out_of_field`(落网格外的触点数)、以及 `WEAK rank-axis`(当顺序沿轴几乎排不出来时显示)。

## 坐标轴口径(重要)

- 横轴 = 沿传播方向位置(归一化,0 = 早端核中心)。纵轴 = 横向位置(归一化,带符号)。
- **横轴/纵轴的归一化分母是"参与触点铺开范围的鲁棒跨度",不是早晚端核间距**——因为真实数据里核间距常远小于触点实际铺开,用核间距做分母会把大半触点推到画面外。物理毫米只作参考层(左上换算 + 标尺),共同比较仍在归一化平面上做。
- 纵轴的正负**只是显示约定,不是解剖左/右**。

## `static_maps/` — 真实病人(52 张,26 个被试,每模板一张)

干净的二维传播正例(早晚组沿横轴分开、横向铺得开、先后顺序沿轴单调):

### epilepsiae_590_t_a.png / epilepsiae_1146_t_a.png / epilepsiae_253_t_a.png
早端一团(紫)和晚端一团(黄绿)沿横轴清楚分开,顺序场有连贯的早→晚梯度,支撑场两个热点对应两团触点。1146/253 的顺序沿轴单调性很强(`rho_x_rank` 0.88–0.92)。
**关注点**:这是这套读出"正常工作"的样子——二维投影合理、支撑 mask 没乱补空区、早端不确定度偏高与"早端是轮流当早端的小群"一致。

### yuquan_chengshuai_t_a.png(1D 对照)
触点几乎排成一条线(单杆主导),标题标 `1D`。顺序仍能左→右读出,但横向是退化的。
**关注点**:1D 被试该进 supplement 而非二维主展示;图正确地把它标出来了。

### epilepsiae_1084_t_b.png(QC 例,弱轴)
触点已收回网格内(`out_of_field=0`),但 `rho_x_rank≈0.05`、标题标 `WEAK rank-axis`——这个模板自身先后顺序沿轴几乎排不出来。
**关注点**:它证明归一化修复避免了"触点全裁到画面外"的灾难,但**不能**当成"有强传播轴"的正例;主展示仍用 590/1146/253。

## `model_maps/` — 模型(3 张,自发活动屏查级产物)

模型输出经虚拟电极写成与真实同构的格式,走**完全相同**的读出。

### model_oneend_pos_t0.png / model_oneend_neg_t0.png(单向行波)
8 个虚拟触点沿一条线早→晚,标题 `1D`、`rho_x_rank` 0.90/0.99。不确定度场几乎全灰(≈0)——模型事件高度可复现,不像真实数据有抖动。
**关注点**:单向行波本就是一维直线传播,被正确标 1D;它的先后顺序场和真实病人的相似度落在真实彼此之间的中上位置(见 archive)。

### model_spont_bidir_t0.png(双向自发)
几何上二维,但 `rho_x_rank=-0.14`、标题标 `WEAK rank-axis`——来回双向传播让早晚混在一起,排不出单一早→晚轴。
**关注点**:**不要因为它"二维"就当成更像真实**;它的顺序场对单个真实传播模板是不匹配的(相似度垫底)。
