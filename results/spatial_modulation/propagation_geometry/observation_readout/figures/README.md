# 传播触点平面读出 — 图说明

这套图把每个传播模板(真实病人 / 模型各一)压成同一张标准化二维触点图,用来比较"模型生成的传播"和"真实病人的传播"在先后顺序与空间铺开上像不像。**所有图只表达传播顺序和采样支撑,不含原始电活动。**

## 每张图的四个子图(主展示布局)

真实病人图每个 subject 一张,固定 2×2 布局:`t_a` 在上、`t_b` 在下;左列是触点散点,右列是平滑顺序场。这样同一 subject 的 A/B 两重传播模板共享同一个 subject-fixed display frame,可以直接上下比较。

1. **左列 — 触点散点**:每个参与触点画一个点,并标出电极名。颜色 = 典型先后顺序(紫=早,黄=晚);点大小 = 该触点参与事件的频率;黑圈 = 这个触点落在临床标注的发作起始区内(**仅叠加展示,不进任何指标**)。
2. **右列 — 平滑顺序场**:把散点的先后顺序按采样支撑加权摊成连续场。标题中标出本图使用的 Gaussian sigma(mm)。这是展示用 smooth,范围比旧版更宽,方便看整体早→晚趋势。

support field 和 uncertainty field 仍在 record / comparison 里保留,但不放入主展示图。它们是诊断层,不是这组图要传达的核心结果。

**标题栏**带四个标签:`1D/2D`(采样是否退化成一条线)、`rho_x_rank`(先后顺序沿传播轴的单调性,越接近 1 越是干净的早→晚轴)、`comparison-grid out_of_field`(归一化比较网格外的触点数)、以及 `WEAK rank-axis`(当顺序沿轴几乎排不出来时显示)。

## 坐标轴口径(重要)

- 图上的横轴 / 纵轴都是 **真实 mm**。同一 subject 的 `t_a` / `t_b` 使用同一个 subject-fixed display frame,所以 A/B 不会各自旋转。
- 这个 display frame 只用于画图;real-vs-model 的场相关仍在 normalized comparison plane 上计算。
- 纵轴的正负**只是显示约定,不是解剖左/右**。

## `static_maps/` — 真实病人(subject-level A/B 合图)

干净的二维传播正例(早晚组沿横轴分开、横向铺得开、先后顺序沿轴单调):

### epilepsiae_590.png / epilepsiae_1146.png / epilepsiae_253.png
早端一团(紫)和晚端一团(黄绿)在真实 mm 平面上清楚分开,顺序场有连贯的早→晚梯度。A/B 两行使用同一视角,所以可以直接看第二模板是同轴反向还是另一路径。
**关注点**:这是这套读出"正常工作"的样子——A/B 视角固定、电极名可核对、SOZ 只作为黑圈叠加。

### yuquan_chengshuai.png(1D 对照)
触点几乎排成一条线(单杆主导),标题标 `1D`。顺序仍能左→右读出,但横向是退化的。
**关注点**:1D 被试该进 supplement 而非二维主展示;图正确地把它标出来了。

### epilepsiae_1084.png(QC 例,弱轴)
`t_b` 触点已收回网格内(`out_of_field=0`),但 `rho_x_rank≈0.05`、标题标 `WEAK rank-axis`——这个模板自身先后顺序沿轴几乎排不出来。
**关注点**:它证明归一化修复避免了"触点全裁到画面外"的灾难,但**不能**当成"有强传播轴"的正例;主展示仍用 590/1146/253。

## `model_maps/` — 模型(3 张,自发活动屏查级产物)

模型输出经虚拟电极写成与真实同构的格式,走**完全相同**的读出。

### model_oneend_pos_t0.png / model_oneend_neg_t0.png(单向行波)
8 个虚拟触点沿一条线早→晚,标题 `1D`、`rho_x_rank` 0.90/0.99。展示用 smooth 范围较宽,用于看整体单向梯度。
**关注点**:单向行波本就是一维直线传播,被正确标 1D;它的先后顺序场和真实病人的相似度落在真实彼此之间的中上位置(见 archive)。

### model_spont_bidir_t0.png(双向自发)
几何上二维,但 `rho_x_rank=-0.14`、标题标 `WEAK rank-axis`——来回双向传播让早晚混在一起,排不出单一早→晚轴。
**关注点**:**不要因为它"二维"就当成更像真实**;它的顺序场对单个真实传播模板是不匹配的(相似度垫底)。
