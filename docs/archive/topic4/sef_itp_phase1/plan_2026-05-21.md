## 3. TDD 测试规格（核心）

每个测试：用 synthetic 输入 → 已知预期 → 断言。**不**用真实 cohort 数据。

### 3.1 H6 测试

#### TT-H6-1: `test_h6_segregation_synthetic_two_clusters`

**Setup**：构造 10 通道 SEEG layout，其中 5 个通道（"A1–A5"）属 shaft A，5 个（"B1–B5"）属 shaft B。设 `participation_rate = [0.9, 0.9, 0.8, 0.85, 0.9, 0.1, 0.1, 0.05, 0.1, 0.15]`，coords 让 shaft A 在 (0, 0, ±2) 一带聚集、shaft B 在 (10, 10, ±2) 一带聚集。

**Expected**：H6 PASS — high-participation 与 low-participation 通道分到两个 shaft（cohort Moran's I 显著正、shaft-stratified null p < 0.05）。

#### TT-H6-2: `test_h6_morans_i_known_autocorrelation`

**Setup**：1D layout 10 通道，participation_rate 单调递增 [0.1, 0.2, ..., 1.0]，相邻通道权重为 1，其他为 0。

**Expected**：Moran's I 接近 +1（强空间自相关），p < 0.001 against permutation null。

#### TT-H6-3: `test_h6_shaft_stratified_shuffle_preserves_per_shaft_mean`

**Setup**：3 shaft，每 shaft 5 channel，已知 participation_rate。

**Expected**：1000 次 shaft-stratified shuffle 后，每 shaft 内的 mean participation_rate 在所有 shuffle 中**完全不变**（因为是同 shaft 内 shuffle）；cross-shaft 关系会变。

#### TT-H6-4: `test_h6_handles_single_shaft_subject`

**Setup**：subject 只有 1 个 shaft 14 channel（real edge case）。

**Expected**：shaft-stratified shuffle 退化为 within-shaft permutation（全脑 shuffle 作为敏感性 fallback），返回 verdict `subject_excluded_single_shaft` 标识。

### 3.2 H1 三层拆分（v1.0.3 修订 2026-05-21 + v1.0.4 framing 收紧）

**为什么拆**：原 v1 plan 把"端点空间紧凑"塞进一个 PASS/NULL 判断里，但其实在问三个不同问题——一个 verdict 撑不起。用户审阅指出 dry-run 上 H1 NULL **不**等于"端点不紧凑"，是"在同电极杆几何已经很紧凑的背景下，额外紧凑性不够强"。把三个问题拆开，每个有自己的判据 + 不同的科学角色。

**v1.0.4 层级假设 framing（用户审阅 2026-05-21 收紧）**：

三层不是平等的"三个 verdict"，是**不同科学角色**：

| 层 | 角色 | 谁能单独证明什么 |
|---|---|---|
| **envelope（包络）** | **falsification gate** — 必要条件 | **PASS** 不证明 mechanism；**FAIL** 是**强反证**（端点跑出参与场就是 framework 危机）|
| **descriptive（描述）** | **几何摘要** — 给读者直觉用 | 报数字 / 画图；**不**单独证明任何 mechanism；**不**进 PASS gate |
| **strict（严格）** | **strong positive evidence** — 真正能升级科学声明的层 | **PASS** = "端点比同杆同参与背景的随机通道更紧凑"，是真阳性；**NULL** = "没看到超过植入几何的额外紧凑性"，**不等于** "端点不紧凑"（descriptive 数字可能仍说明端点紧凑——只是不超过 baseline）|

**关键纪律（v1.0.4 lock）**：

- **`strict NULL` 永远不能翻译为 "端点不紧凑"** —— 它只说明"没看到超过植入几何 baseline 的额外紧凑性"。cohort 描述层数字才告诉你端点本身是不是空间靠近
- **`envelope` 是 falsification gate，不是 PASS 装饰**：envelope PASS 不增加 mechanism 证据；envelope FAIL 否定整个 framework
- **`descriptive` 数字必须独立报告**：不可只报 strict PASS/NULL 不报描述层；下游读者必须能看到"端点实际距离多少 mm"
- **overall verdict 必须包含 envelope 状态**：envelope SKIPPED → overall = `INCOMPLETE_GATED_ON_COORDS`（v1.0.4 src 已实现，禁止 envelope SKIPPED 时给 PASS / NULL / partial_PASS）

**期望产出（advisor 2026-05-21 #3 expectation-setting）**：

- **H1 strict PASS 预期 cohort 上是少数案例**——同杆 matched null 的几何 baseline 已经很紧凑，"额外紧凑性"门槛高
- **H1 主交付不是 strict PASS**，是 **envelope（falsification gate）+ descriptive（绝对几何数字）**；strict 作为强阳性证据的上界
- cohort 写作时不要把"H1 strict cohort NULL"叙述成阴性结果——要写成"端点紧凑性与同杆 baseline 一致；descriptive 显示端点实际距离 X mm，envelope 全 cohort PASS（端点未脱离参与场）"

#### H1 描述层 (h1_descriptive)

**朴素问法**：端点在空间上靠近吗？

**怎么算**：直接报数字，**不**做 null 比较，**不**给 PASS / NULL / FAIL：

- `mean_pairwise_distance(endpoint_set)`
- `diameter(endpoint_set)` = max pairwise distance
- `radius_of_gyration(endpoint_set)` = sqrt(mean(|x_i - centroid|²))

**用途**：cohort 上报描述统计、画分布图、给读者直觉。**不**进任何 PASS gate。

#### H1 严格层 (h1_strict)

**朴素问法**：端点比"同电极杆、同参与背景下的随机通道"还紧凑吗？

**怎么算**：matched random null with shaft + participation + HFO rate（与原 v1 一致），degradation order 不变。检验 `C_actual < C_null`。

**用途**：**这是真正的 PASS gate**。但 framing 要清楚——这条问的是"**超过植入几何背景的额外紧凑性**"，不是"端点是否紧凑"。

**verdict 解读**：

- **PASS**：端点比同杆背景下的随机通道更紧凑，存在 extra-compactness beyond shaft geometry
- **NULL**：端点紧凑性与同杆背景一致 —— 不意味着端点不紧凑，只意味着"杆几何能解释观察到的紧凑性"
- **FAIL_DIFFUSE**：端点比同杆背景更分散 —— 比 NULL 更糟，但仍不否定 mechanism（可能只是端点提取不稳）

#### H1 包络层 (h1_envelope)

**朴素问法**：端点有没有跑出整体病理参与场？

**怎么算**：端点到 non-endpoint 参与通道 centroid 的距离 / non-endpoint 自己到该 centroid 的距离（advisor 2026-05-21 fix：centroid 不含端点）。

**用途**：这是**必要条件**，不是充分条件：

- **PASS**：端点在参与场内 —— **不**单独证明 "紧凑机制"，只说明端点没离谱
- **FAIL**：端点显著跑出参与场（ratio >> 1） —— **严重**问题，触发 framework §4 失败模式（"endpoint 脱离参与场"）
- NULL 不适用（必要条件只有 PASS / FAIL）

#### H1 整体 verdict 规则（修订）

```
if h1_envelope == FAIL:
    H1 整体 = FAIL   # 必要条件失败覆盖一切
elif h1_strict in {PASS, partial_PASS}:
    H1 整体 = h1_strict.verdict
else:
    H1 整体 = NULL
```

**关键纪律**：三层都报告，cohort summary 写"H1 描述层中位 X mm，H1 严格层 cohort Wilcoxon p=Y，H1 包络层 cohort N PASS / 0 FAIL"。**禁止**只报一个 H1 verdict 把三层揉掉。

### 3.3 H2 测试

#### TT-H2-1: `test_h2_set_reversal_perfect_swap`

**Setup**：`S_A = {0, 1, 2}`, `K_A = {7, 8, 9}`, `S_B = {7, 8, 9}`, `K_B = {0, 1, 2}` （完美 swap）。

#### TT-H2-3: `test_h2_spatial_reversal_two_blob_synthetic`

**Setup**：endpoint 落在两个 spatial blob — blob A coords ≈ (0, 0, 0), blob B coords ≈ (10, 10, 10)。`S_A` 在 blob A，`K_A` 在 blob B，`S_B` 在 blob B，`K_B` 在 blob A（perfect spatial reversal）。

```
  └─ event_metadata 或 raw lagPat npz → 派生 eventsBool (n_ch, n_ev)
  └─ (Phase 0b 后) 3D coords (n_ch, 3)  ← 当前用 shaft-ordinal 替代

中间步骤:
  1. eventsBool → participation_rate (n_ch,)
  2. channel_names → shaft groups via _extract_shaft_prefix
  3. (Phase 0b 后 OR shaft-ordinal 主分析):
     - distance matrix (n_ch, n_ch) = shaft_ordinal_distance(...)
  4. compute_h6_segregation(participation_rate, coords_or_shaft, shafts)
     - 输出: morans_i, centroid_distance, silhouette + null p-value

输出: per_subject_h6_<sid>.json
```

### 4.2 H1 数据流（依赖 Phase 0a + Phase 0b）

```
输入:
  - per_subject masked JSON
  - PR-6 endpoint output on masked @ results/interictal_propagation_masked/pr6_*/  ← Phase 0a §5f 完成后
  - coords (n_ch, 3)  ← Phase 0b 完成后

中间步骤:
  1. load PR-6 endpoint set per cluster: {S_A, K_A, S_B, K_B}
  2. (per cluster) compute_h1a_within_source(S, coords, ...)
  3. (per cluster) compute_h1b_within_sink(K, coords, ...)
  4. (per cluster) compute_h1c_envelope_within_field(E, all_valid, coords)
  5. integrate H1 verdict per subject per cluster

输出: per_subject_h1_<sid>.json
```

### 4.3 H2 数据流（依赖 Phase 0a + Phase 0b）

```
输入: 同 H1 + Tier 1+ sub-cohort 资格筛选 (non_endpoint_participating ≥ 4)

中间步骤:
  1. load fwd/rev pair from PR-6 masked
  2. extract (S_A, K_A) from cluster A, (S_B, K_B) from cluster B (paired)
  3. compute_h2_set_reversal(S_A, K_A, S_B, K_B)
  4. compute_h2_spatial_reversal(S_A, K_A, S_B, K_B, coords)
  5. if Tier 1+: compute_h2_pca_sensitivity(...)

输出: per_subject_h2_<sid>.json
```
