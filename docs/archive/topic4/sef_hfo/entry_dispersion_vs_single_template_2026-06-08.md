## 摘要（三段式朴素话）

**测了什么.** 我们想验证一个说法能不能从「**间期 HFO 群体事件由两个固定端点反复点火**」改写成「**事件每次进入同一条有抖动的中观刻板传播通路，早端是一小撮通道轮流领先**」。要分清这两种图景，我们在每个被试、每条稳定传播模板里看两件事：每次事件**最先亮起来的那个通道**，是死死钉在一个点上，还是能在一小撮通道里换人（这次 A、下次 B）；以及它**后面那串通道的先后顺序**还稳不稳。（"最先亮"指的是这次事件里最早被检出、参与进来的通道——它只是**到得最早的那个**，不等于病灶源头。）

**怎么测的.** 这里有个陷阱：**哪怕只有一条固定传播路径、每次只带一点随机抖动，也会自动长出「第一个通道分散 + 后面顺序稳定」这个样子**——因为「谁排第一」恰好是整条顺序里最经不起抖动的位置（一丁点噪声就能让第一名换人），而后面的顺序照样稳。所以原本设想的判据「第一个分散 + 后面稳 → 有一片独立入口结构」根本分不开「独立入口结构」和「一条带抖动的单一通路」；拿它去比「固定单点（第一名永远同一个）」或「完全随机打乱」也都不承重。

真正能分开的对照是这样：照这个被试这条模板里**每个通道的平均先后位置**、再加上**实测到的抖动幅度**，造一大批「只有一条路径 + 抖动」的假事件（参与的通道跟真实事件一模一样），然后用完全相同的方法量「第一个通道有多分散」。只有当**真实数据里第一个通道的分散程度，比这批「单一路径+抖动」假事件还大**，才说明存在一条单一通路之外的额外入口结构。我们对每条模板都跑这个对照，并辅以两个互补的看法：(1) 第一个通道的出现频率是不是就顺着模板「从早到晚」单调衰减（单调衰减 = 一条带抖动的通路的低端尾巴；如果在模板偏晚的通道上冒出第二个峰，才是另一个独立入口峰）；(2) 不管这次是哪个通道先进来，后面的通道顺序是不是都一样（都一样 = 大家最终汇进同一条下游通道）。

**揭示了什么.** 一句话：**「两个固定端点反复点火」可以放心否掉——改写成「事件反复进入一条有抖动的刻板传播通路、早端是一小撮通道轮流领先」站得住；但「这一小撮入口比单一带抖动通路本身更分散（构成一个独立入口结构）」这个更强的说法在这套数据里立不住。**

(1) **最硬的结论——入口几乎从来不是一个固定点。** 95 条模板里 82 条（86%）的最早通道铺在不止一个通道上（出现频率最高的那个入口通道，中位也只占了 37% 的事件）。所以「每条模板只有一个（正反两端各一个）固定触发点」这个严格图景被否掉，且不依赖任何 null 的选择。**口径要放稳，别夸大**：模板本来就是统计性的（簇内顺序一致性 τ 中位 ~0.25，从来不是 1.0），所以「入口不是单点」很大程度是在**量化「入口是这条已然松散的模板里最不受约束的那个位置」**，而不是推翻谁强烈主张过的模型；而且这份分散有一部分来自**采样**（每次哪 3–4 条通道越过检出阈在变，最早自然跟着变），并不等于解剖意义上的入口可移动性。

> **不要串台（§6.2/§6.3）**：这里说的「端点」是**每个事件实际最早被检出的到达通道**（per-event arrival leader）。它**不是** PR-6 的 source/sink **端点锚定**——后者是从**模板均值名次**取的固定 top/bottom 集、做的是解剖 SOZ 富集。本结论**不触及、也不推翻** PR-6 模板级端点锚定结果，两者是不同的对象。

(2) **更强的「入口分散超出单一通路」说法——没看清，而且很可能是假象。** 「真实入口比单一带抖动通路还分散」的比例，随 null 的严格程度一路缩水：宽松（同质抖动）null 下 49/95（52%），换成保守（每个通道用自己噪声幅度）null 只剩 34/95（36%），两种 null 都超过的「稳健」子集只有 27/95（28%）。更关键的是：这些「稳健超过」的簇恰恰是**通道最多、聚类最松**的那批（簇内顺序一致性中位 0.28，而其余簇 0.40；通道数中位 10 vs 8；最极端的是 52 通道、簇内一致性低到 0.11 的被试）；反过来，**判为「比单一通路还集中」（偏端点）的，全是通道少、聚类紧的小montage**（中位 6 通道、一致性 0.47）。也就是说，「入口更分散」基本是「这个簇本来就是松散混合 / 一条被切两半的连续谱」机械造成的——多通道里取最早，本来就更容易换人。我们的检验**能否定「干净的单一模板」、却不能确认「独立入口结构」**（紧凑的近似并列入口会被 null 复现）；所以对这个更强说法只能说**看不清、且证据更偏向假象**。

(3) **综合画面**：数据落在「严格固定端点（已否）」和「入口分散超出单一通路（确认不了）」之间，**最省事的解释是「每条模板=一条有抖动的中观刻板传播通路，早端有一小撮通道轮流领先」**：最早通道是这条顺序里最受抖动影响的位置（所以会换人），去掉最早通道后剩下顺序的刻板度几乎不变（centered τ 变化中位仅 0.03），下游顺序基本不随「谁先进来」改变（但这条**不**能区分吸引子和单一模板，两者都给同样结果，仅作描述）。**不要**据此写「独立入口结构 / 通道吸引子被证实」。

## 1. 为什么原 handoff 判据不承重（方法学核心）

原 handoff 的成功判据是「最早通道分散 + 后续排序稳定 → 支持独立入口结构 + 通道吸引子」。它不成立的原因，是一个统计事实：

- 「最早通道」是整条排序里的**极值**（取最小先后位置的那个通道）。极值对噪声最敏感——同一条固定模板，只要每次加一点点抖动，最早名次就会在早端的几个通道之间跳来跳去。
- 因此「最早分散」+「后续稳定」是**一条带抖动的单一模板的自然产物**，不是独立入口结构的证据。
- 拿它去比「固定单点」（分散度≈1）或「随机打乱排序」（后续稳定度≈0）这两个**稻草人** null，必然轻松「通过」，但什么都没证明。

簇内约 92% 的排序一致性本来就来自**固定的通道身份**（Topic 0 §3.1 / Topic 1 §3.2：raw τ 大头是通道身份排序，centered τ ≈ 0.03），所以「后续排序 vs 随机」这条腿几乎必然通过、**不承重**。本分析因此把它降为描述层，主判据换成下面的生成式对照，并要求该检验**可以**返回 NULL（本仓库奖励诚实 NULL）。

## 2. 承重检验：单一路径 + 实测抖动的生成式 null

- **模板** `t[ch]`：该模板簇内所有事件中，每个通道**归一化先后位置的均值**（只用参与通道；先后位置用 phantom-safe 的 masked 重排，避免不参与通道的假名次污染）。
- **抖动**：把每个事件里「实际归一化位置 − 模板位置」的偏差**汇成一个池子**（跨通道合并 = 同质「一条模板 + 共同抖动」null）。一个通道如果有**真正的第二入口模式**（在某些事件里跳到最前、超出它自己的抖动幅度），这个偏差就**不会**被同质 null 复现 → 可被检出。
- **每次重抽**：`score[ch] = t[ch] + 重抽一个偏差`，在**与真实事件相同的参与通道**里取最早 = argmin；重算「有效入口通道数」（= exp(香农熵)）和（有坐标时的）入口空间半径。重复 500 次得到 null 分布。
- **方向性 p**：`p_excess = P(null ≥ 观测)`（观测**更分散** = 独立入口结构证据）；`p_concentrated = P(null ≤ 观测)`（观测**更集中** = 偏固定端点）。

单元测试（`tests/test_propagation_entry_dispersion.py`，7 项全过）锁住两条关键性质：(a) 把数据造成「一条模板 + 对称噪声」时，该 null **不会**误报 excess（观测落在 null 带内）；(b) 当某个通道领先得超过抖动允许时，null 正确判定观测**更集中**。

> **该 null 的灵敏度边界（诚实声明）**：因为 null 重抽的是**边际**偏差，它会吸收掉大部分边际分散度。所以「有效入口通道数」这条 excess 检验是**保守的**（不易被超过）；它对「比单一模板更集中」（端点倾向）很灵敏，对「更分散」相对钝。真正分辨「另一个独立入口」的力量更多在**入口形状**（频率是否在模板偏晚处冒第二个峰）和**空间半径**上，「下游是否随入口改变」则独立判定「单一下游通道 vs 多条入口依赖的路径」。三者并报。

## 3. 队列与口径

- 队列 = `results/interictal_propagation_masked/per_subject/` 里 `chosen_k ≥ 2` 的全部被试（40 个：20 Epilepsiae + 20 Yuquan；其中 34 个 `stable_k = 2`，是 framework 主队列，另 6 个高 k）。**每条模板簇分别算，正反模板不合并。**

## 4. 全队列结果

队列 = 40 个被试 / 95 条模板簇（20 Epilepsiae + 20 Yuquan，全部 `stable_k` 真聚类；每条簇 ≥50 事件才算 null）。数据：`results/propagation_entry_dispersion/cohort_summary.{json,csv}`。

### 4.1 最硬结论：入口不是单一固定点（不依赖 null）

| 指标 | 值 |
|---|---|
| 入口铺在 >1.5 个有效通道上的簇 | **82 / 95（86%）** |
| 最频繁入口通道占事件比例（中位） | **0.37** |
| 入口有效通道数（中位） | 5.4 |

→ 严格「两个固定端点反复点火」被否掉。**但口径放稳**：模板本就统计性（簇内 τ~0.25≠1.0），这条主要是**量化「入口是已松散模板里最不受约束的位置」**、且部分来自检出采样（哪几条通道越阈在变），不是戏剧性翻案，也不是解剖入口可移动性的证据。**注意不要串台**：此处「端点 = 每事件实际最早到达通道」，**与 PR-6 模板均值名次取的 source/sink 端点锚定（解剖 SOZ 富集）是两个对象，本结论不触及 PR-6 端点结果**。

### 4.2 「比单一路径更分散」=null 依赖的 bracket（不承重）

| null | excess 簇数（α=0.05） | 比例 |
|---|---|---|
| 宽松（pooled 同质抖动） | 49 / 95 | 52% |
| 保守（per-channel 各自噪声幅度） | 34 / 95 | 36% |
| **两种都超过（稳健）** | **27 / 95** | **28%** |

四档 verdict：concentrated（比单一路径更集中/偏端点）35、consistent（与单一路径兼容）11、robust_excess 27、fragile_excess（仅宽松 null 超过）22。

### 4.3 混淆交叉表：「稳健超过」追着「簇松散 + montage 大」跑

| 档 | n | 簇内顺序一致性 τ（中位） | 通道数（中位，范围） |
|---|---|---|---|
| concentrated（偏端点） | 35 | **0.47**（最紧） | **6**（4–26） |
| consistent（单一路径） | 11 | 0.30 | 8 |
| fragile_excess | 22 | 0.29 | **15** |
| robust_excess | 27 | **0.28**（最松） | 10（4–**52**） |

→ 「入口更分散」与「簇松散 / 通道多」强烈共变：robust/fragile-excess 是松散大montage（τ≈0.28、通道 10–15，含 52 通道、26 通道、24 通道等被试），concentrated 是紧凑小montage（τ=0.47、6 通道）。`is_mixture`（pairwise-τ 多模态旗标）对**所有** 95 簇都为真，不能区分。这正是 Topic 1 §3.1d 的混合/连续谱审计预言的方向（442 = UNIMODAL 连续谱被切两半，正落在 fragile_excess；958 = BIMODAL，落在 robust）。所以 28% 这个「稳健超过」不能读成生理上的独立入口结构——它机械地由聚类松散度 + 通道数驱动。

### 4.4 分队列 + 辅助描述

- 分队列：Epilepsiae 44 簇（robust 11 / fragile 8 / concentrated 19 / consistent 6）；Yuquan 51 簇（robust 16 / fragile 14 / concentrated 16 / consistent 5）。Yuquan 的大montage（接 broad-lagpat / 多杆）贡献了较多 robust。
- 入口形状：最早频率 vs 模板先后位置的 spearman 中位 **−0.79**（多数仍是「单一模板早端的噪声尾巴」单调衰减，而非模板偏晚处冒第二个峰）。
- 去掉最早通道后 centered τ 变化中位 **0.027**（最早通道**不是**刻板度的锚；下游顺序自带稳定）。
- 下游随入口不变：cross-entry 一致度中位 **0.80**——但此指标**不**能区分「吸引子」与「单一模板」（两者都给≈1，只在「入口决定下游路径」时才掉，而稳定簇内不会发生），仅作描述，不承重（advisor 2026-06-08）。
- 空间半径：宽松 null 下 55/69 簇观测入口半径 > null（80%），但这是 pooled-only + 同样受异质性/montage 混淆，仅描述，不进结论。

### 4.5 这套检验能/不能回答什么（诚实边界）

- **能**：否掉「单一固定点点火」（入口铺开，4.1）；否掉「干净单一模板」当某簇的入口确实更分散时（4.2 保守 null）。
- **不能**：确认「独立入口结构」——紧凑的近似并列入口会被 null 复现，且「更分散」与「簇松散/连续谱/montage 大」分不开（4.3）。也**不能**把「下游随入口不变」当吸引子证据。
- 一句话：**reject 比 confirm 强**。主叙事可以从「两固定端点」松绑成「一条有抖动的刻板传播通路 + 早端小入口群轮流领先」，但不能升级成「独立入口结构 + 通道吸引子被证实」。

**下一步（能 confirm「独立入口结构」的检验，本轮未正经做）**：真正能把这个更强说法从「分不开」推到「测过了」的，是**入口集合相对该模板整条传播范围的空间紧凑度**——最早通道们是聚在一小片局部子区域里（= 真有一个局部独立入口结构），还是散在整条通道上（= 只是松散模板的产物）。本轮只算了 pooled-only 的空间半径（已降级为描述，且同样受异质性/montage 混淆），**没有**做「相对整条传播范围、且按坐标覆盖率门控」的紧凑-子区域检验。把它列为明确的下一步，则「独立入口结构」应表述为**「尚未正经测过」**，而非仅仅「原理上确认不了」。

**一处口径不对称（caveat）**：`excess` 是用宽松+保守两种 null 夹出来的，但 `concentrated`（偏端点）目前只看了宽松（pooled）null。两边严格度不完全对齐；`concentrated` 计数应理解为「相对同质单一模板更集中」，未单独过保守 null。

### 4.6 「小入口群」与「两条通路的入口相反」（2026-06-08 追加描述）

承接上面的图景，补两个直接刻画「早端小入口群」的描述性数字（`results/propagation_entry_dispersion/cohort_summary.csv` 的 `obs_earliest_prob`）：

- **早端是一小撮通道，轮流领先**：单个最常见的入口通道中位只占 37% 的事件（不是单点）；但**前 3 名入口通道累计覆盖中位 74%** 的事件（55% 的模板里前 3 名 ≥70%）。也就是说「这次 A、下次 B」换的是**一小撮（约 3 条）通道**，它们整体稳定地待在早端。**注意**：这「一小撮」本质上就是平均名次最靠前的那几条通道（= identity bias 的早端），并**不是**一个独立于通路的入口子系统；而且「A/B 换人」有一部分只是**检出采样**（这次 A 越阈、B 没参与），不一定是动力学竞争。
- **两条通路（正反模板）从相反的小入口群进入**：在 34 个 k=2 被试上，两条模板的「各通道当最早的频率」向量呈负相关（Spearman 中位 **−0.49**，76% 为负），且 **100%** 的被试两条模板「最常见入口通道」不同。即模式 A 从一小撮通道进、模式 B 从大致相反的另一小撮进——与「两条互为反向、共享同一条空间轴」一致（一条的早端正是另一条的晚端）。

**口径**：k=2「两种时序模式」本身是上游逐事件聚类（PR-2）的结果，**不是**本入口分析独立重新发现的；本分析只是确认「入口尊重并复现了这两模式，且两模式入口相反」。「只用入口（不看后续顺序）能否独立聚出两模式」未做。
