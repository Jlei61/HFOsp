## 0. 朴素话三段

**测了什么**：把"戳一下→沿连接方向传播→虚拟电极读方向"这套，在能插真实 4mm 电极的厘米尺寸放电网络上，
看四个对照是否都对：(C1) 连接方向取 0/45/90° 读出是否跟着转；(C2) 踢点挪位是否把方向带跑；
(C3) 电极杆整体旋转是否把方向带跑；(C4) 各向同性连接（无方向）是否如实读不出稳定方向。

**怎么测的**：把厘米尺度的放电网络单端戳一下，它沿连接方向推出一道**持续行波**（不是自限的小团）。
读"第一遍前沿扫过各触点的先后"（=传播方向），用早到→晚到 centroid 的连线作方向估计。两种读法都做：
神经元发放包络 + 电流型 LFP（更接近真实电极录到的波形）。同一套几何/估计子/阈值，只换基底（rate→SNN）。

**揭示了什么**：厘米尺度放电网络读出的方向**确实来自模型的连接，不被电极摆位、踢点位置、脚本带跑**。
在内存能跑下来的 L=20mm 大片（密度调低到 N≈40k）上，**四个对照在同一个工作点全部过**——连接转方向跟着转
（C1），踢点横挪方向不变（C2），电极杆转方向不变（C3），各向同性连接读不出稳定方向（C4，可读性塌到 0.02）。
两种读法（发放 + 电流-LFP）一致。**先前在 12mm 小片上 C2 没过，是因为片子装不下足够大的电极阵去接住被横挪
的波——换成大片就好了，不是模型坏。**

## 1. 关键 reframe + scale law（用户驱动）

**reframe (option A)**: cm 尺度 SNN 单端踢出来的是**持续推进的行波**（不是小尺度的自限团）。用"等事件
自终止"的窗（event_window_for_run）读 → n=0（波不自终止）。改读**第一遍前沿**（first-crossing onset，
固定窗），估计子不变（WF-A 垂直陷阱只坑 onset_front_axis，不坑 endpoint_centroid_axis）。

**scale law（验证后直接驱动调参，用户 2026-06-08 提示）**:
1. **mean-field 工作点对 N/density 不变**（固定入度 C_EE=800）→ g=3.6/drive=0.6 **跨尺度不重调**。
2. **有限尺寸噪声 ~1/√N**：小 N 噪声大→事件自限；大 N 噪声小→行波持续 + 周期边界 re-entry。
3. **前沿速度 v≈0.13–0.22mm/ms 是 intensive**（强度量）→ 读窗随 L 缩，不随 N。
4. **建连内存 ∝ density×L²（= N），不是单看 L**（2026-06-09 修正）。OOM 是被**神经元数**卡死：
   L=12 density=1000=144k 建连 OOM>215GB；但 **L=20 density=100=40k 只占 13 GB（比能跑的 L=12/d500=72k
   还小）→ 大片完全可行**。density 是降 N 的旋钮（不动工作点）；低密度同时让有限尺寸噪声把行波拉回自终止。
   **此前"N² 内存墙挡住 L≥20"的归档说法已撤回**（见顶部修正框 + §2 C2 闭合）。

**读法修复链**: 早踢（t_kick=20，默认 150 太晚→短 T 把前沿切断误判 runaway）+ 固定第一遍窗 +
endpoint_centroid_axis + 低密度大片。

## 3. 工程教训（跨夜自动扫失败,已修）

跨夜 14-条件自动扫**零结果**: ①raw `nohup python &` 后台没扛过会话中断（`/compact` API 错）→ 5min 死；
②内存看门狗 `pgrep -f run_sef_hfo_snn_cm_explore` **匹配到自己的命令行**（命令里含该串）→ 自检失明,
进程死了没察觉没重启。**修复**: harness `run_in_background`（持久）+ **PID-file 看门狗**（按进程号杀,不按名）
+ 脚本内 `/proc/meminfo` 自检（<40GB 优雅退出）。

**第二个工程教训（2026-06-09）**：内存约束要写成 **N=density×L²**，不要写成"L 的墙"。把约束归因到错的变量，
直接导致"C2 不可闭合"的错误结论被写进归档。正确的旋钮是**降密度换大片**（mean-field 工作点对密度不变，
scale law §1.1），低密度还顺带把行波拉回自终止。

  这与 rate runner-v2 的四对照一起 = **双模态方向读出 parity 闭合**。
- **L=12 审计版（§2.2）保留为证据**：它的价值是 (a) valid-mask 抓出 montage 出片污染（C2 LFP 9→6）；
  (b) 暴露"片小→大电极阵装不下→C2 覆盖不足"这条线索，正是 L=20 闭合的动机。
- 阈值/估计子/几何/D6 未动。工作点 g=3.6/drive=0.6 跨尺度不变（scale law §1.1）。

### 验收（用户问：能否在 cm 尺度 SNN 上做 kick + 噪声驱动的异质性实验）

- **kick 驱动的 cm 异质性实验：读出层已验收（YES）。** 在 cm 尺度真实 4mm 电极上"戳→沿连接传→读方向"
  四对照全过；放一个病理核、单端踢、读方向，这套读出是可信的。**唯一未做的是"把异质核搬上 L=20 大片
  真跑一遍"**——读出层先验过了，cm 病理核的集成跑是下一步的实验本身（小尺度 L=3mm 异质核已验过机制螺丝，
  cm 集成未跑）。可以开跑。
- **噪声驱动的 cm 异质性实验：还不能验收（NOT YET）。** 噪声/自发点火这条线目前是**单次实现的"风险警报"
  级别**（wide-everywhere → 328Hz 自发爆发，需 multi-seed + finite-size + density 扫才能升级为机制），
  且同质噪声扫已被降级。到了 cm 尺度更纠缠：让 cm 事件自终止、可读的正是"有限尺寸噪声"——而它本身就是
  噪声驱动实验要研究的对象（循环）。**先在小/中尺度把自发点火 regime 定下来，再上 cm。**
- **路线图张力（诚实标注）**：异质性主线的下一步本来就锁"NOT cm-scale first"——验证过的关键路径是
  小尺度异质核 + 前向 LFP 读出 + B-side multi-seed。**所以 cm 方向读出现在是一个"已单独验收、随时可用"的
  能力，但 cm 异质性实验目前不在直接关键路径上。**
