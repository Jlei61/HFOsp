## Take-home（第一性原理，先看这段）

**测了什么。** Step-1 在简化的"平均放电率场"上得到一个诚实空结果：把一片**完全均匀**的组织、加上慢的、空间相关的随机噪声，扫遍"驱动强度 × 噪声强度"整个面，**找不到一块稳健的、噪声自发冒出"一个一个、放完就灭"的群体事件、且事件稀疏到对得上真实数据的区域**——所谓"离散"只是静默和持续之间一道靠运气挑种子才出现的窄缝。我们现在把同一个问题搬到**真·一个个神经元放电**的网络里问：放电网络是不是也这样？

**怎么测的。** 同质放电网络（所有神经元参数全同），加一个**慢（~150ms）且空间相关（~0.4mm）的噪声场**到兴奋神经元的外部输入上（直接复用 rate 场那套噪声生成器；放电引擎原装的噪声是个"全片一起亮"的标量噪声，点不出局部事件，所以必须换成空间噪声才算忠实对照）。在"驱动 × 噪声强度"网格的每一格、每个随机种子，量一条**空间相干的群体活动曲线**，用**与 rate 场完全相同的离散事件判据**（每一段越阈都必须自己回落=自终止；活动占时>30%判为持续；事件率必须落在 [0.01,1]/秒 的数据兼容带）判定，并要求 ≥60% 种子离散才算这一格"有事件"。判定是**拓扑性**的——问"整个面里**有没有**一块稳健的数据兼容离散区"，而不是"在和 rate 场相同的噪声值处"（两个模型的噪声单位不可公度）。

**揭示了什么（降级后，诚实口径 — 见 §7）。** 站得住的只有两点：(1) 不给噪声时网络每个驱动都**静默**；(2) 给同质组织加慢空间噪声，它**很容易、很稳健**地自发出"一个一个、放完就灭"的离散事件（drive≥0.6、各噪声档几乎都 3/3 种子）——这点机制上恰与 rate 场相反（rate 场是"离散事件几乎压根冒不出来"）。**之前据此下的"SNN 也没有数据兼容离散区、复现 rate 场 NULL"已撤回**：唯一依据是"事件率 2–8/秒、太快"，而这个比较不可靠——真实数据率（每 1–100 秒 1 个）数的是**穿出来、被多通道捕获的群体事件**，模拟里数的是片子里**每一个**相干自终止成核（很多局部成核在真组织里穿不出来/检不到），分母不同；加上每条只跑 1.5 秒，分辨不了 [0.01,1]/秒 的低频端。所以**"事件是否太快、同质组织够不够、是否复现 NULL"目前无法判定**。**"需要病理热点"只能作下一步假设**，且它更硬的理由是**模板多样性**（同质 → 四处随机成核 → 模板多 → 与数据 ~2 矛盾），不是事件率，而本测试**没测模板多样性**。

## §1 问题与对照基准

- **基准（rate 场 §11）**：同质 window-A 率场，(drive×σ) 联合 + 逐点重标检测器 → **诚实 NULL**（0 accepted 格、`robust_2d=False`，σ=0 每 drive extinction）。离散只是 silent↔sustained 间一道种子脆弱窄缝。
- **本测试的问**：放电真值底物上，同质组织 + 慢空间噪声，是否存在稳健的数据兼容噪声自发离散事件区？
- **存在性的去向（不是本测试的范围）**：若同质也 NULL → 把存在性移交"异质核"（Step 3 病理→参数映射 spec）。本测试只判同质底物。

## §2 方法（advisor 2026-06-05 加固的三条忠实性 + 复用）

1. **噪声 = 慢 + 空间相关**：复用 `src.sef_hfo_events.make_ou_noise`（τ=150ms、ell=0.4mm≈l_EE），加到 E 神经元外部速率。**放电引擎原装 OU 是标量/全局**（`model.py`/`kick_probe.py` 的 `xi` 广播全网）→ 只能整片同亮、点不出**局部**事件 → 用它得 NULL 是"全局噪声点不出局部事件"的错误理由。故加空间噪声场。
2. **观测量 = 相干 active fraction**：每 2ms bin 内每格的局部 E-active 比例 → 空间平滑（ell≈l_EE）→ 越过固定结构性 crossing（0.04）的格子占比。**不是** blind 的逐神经元计数（散点 speckle 会虚高）。
3. **探测器 = 原样复用** `classify_run`/`calibrate_detector`/`accepted_cell`（同一离散合同）：all-returned、frac_time_on≥0.30→sustained、每点幅度 bar = σ=0 地板↔σ=0 kick 峰值中点、accept 率∈[0.01,1]/s。
4. **σ=0 控制**：每个 drive 的 σ=0 不踢 run 必须 extinction（无噪不自发）；σ=0 踢 run 提供 bar 上参考（kick=真事件峰值）。
5. **拓扑判定**：稳健二维区 = ≥2 个 accepted 格；σ_n（1/ms on Poisson）与 rate 场 σ（mV on muE）**不可公度**，不在"相同 σ"处读判定。

**尺度**：L=3mm、density 1800、N≈16200、T=1500ms。（L=1 整片秒亮、L=2 事件近全局 → coh 饱和到 1.0 无动态范围；L=3 事件 sub-global，coh 有动态范围——见 §3。）

## §3 控制结果（PASS，且定标了尺度）

| 控制 | 结果 | 判定 |
|---|---|---|
| σ=0 不踢（每 drive） | max_coh=0, rE_max≈3Hz | **extinction ✓**（无噪不自发） |
| σ=0 + kick（drive 0.6） | max_coh=1.0, → `discrete_events`（1 个自终止事件） | **wiring + 事件清 bar ✓** |
| L=2 vs L=3 | L=2 coh 饱和 1.0（事件近全局）；L=3 kick_peak=1.0 但噪声事件 sub-global | **选 L=3** |
| drive=0.5 kick_peak | 仅 0.0069（近 fizzle，远低于 0.6+ 的 1.0） | drive 0.5 在 Hopf 下很深、近不可激 |

（粗体率 = 离散但率>1/s 超数据带；✅ = 唯一 accepted 格）

**两个稳健观察（与 tau 无关）**：
- **σ=0 每 drive 都 extinction** —— 无噪不自发，控制守住。
- **离散性本身稳健而普遍** —— drive≥0.6、frac≥0.02 几乎都是 3/3 `discrete_events`（自终止、all_returned）。即**同质放电网络确实能从慢空间噪声里反复自发冒出离散自终止事件**。这与 rate 场"离散几乎不存在"在**机制上相反**。
- **唯一卡住的门是"率太高"**：除 frac≈0.02 的近阈窄沿外，事件率 2.4–8.4/s ≫ 数据带 [0.01,1]/s；唯一 accepted 格（drive 0.6/frac 0.02，2/3 seed）本身脆弱。

## §5 tau 敏感性扫描（承重 blocker，进行中）

**为什么 blocking（advisor 2026-06-05）**：§4 里**每一个非 accepted 格都只栽在率门上**，离散门几乎处处通过。整个 `robust_2d=False` 全靠"率 2.4–8.4/s 超 [0.01,1]/s"这一个数，而这个率正可能由我固定的噪声时间常数 TAU_NOISE=150ms 设定，不是网络内禀极限。两种机制预测相反的 τ-依赖：
- 率由**噪声越阈统计**定 → 慢 τ → 越阈更稀 → 数据兼容带变宽 → **结论反转**（"同质组织能稳健出离散事件"）。
- 率由**事件时长 + 网络恢复**（GABA 18ms、不应期）定 → 慢 τ 不怎么改高 frac 的率 → 该率是**网络内禀**（不由噪声时间常数设定）。**注意（§7 #5）**：tau 扫描只能确立"率内禀、非 tau artifact"这一点；它**不**确立"内禀率太快、对不上数据"——后者是被撤回的分母错配比较，tau 扫描管不到。

**扫描**：drive=0.6、tau∈{150,300,600,1000}ms、frac∈{0.05,0.10}（仅栽于率的格）、seed 1/2/3。
数据：`snn_noise_tausweep.json`（+`.log`）。

**结果（n_accepted=0，率对 τ 不敏感）**：

| frac | τ=150ms | τ=300ms | τ=600ms | τ=1000ms |
|---|---|---|---|---|
| 0.05 | 3.6/s (3/3) | 3.8/s (3/3) | 2.9/s (3/3) | 2.9/s (3/3) |
| 0.10 | 4.7/s (1/3) | 5.8/s (3/3) | 6.3/s (2/3) | 5.1/s (3/3) |

把噪声放慢 6.7 倍，事件率**纹丝不动**（frac=0.05 始终 ~3/s，frac=0.10 始终 ~5–6/s），**从未接近** [0.01,1]/s 上限。→ 率**不是**由噪声越阈统计设定（那应随 τ 变稀），而是由**网络内禀事件时长 + 恢复**（GABA 18ms + 不应期）锁定。advisor 预测的机制 #2 命中。

## §6 判定 — 初步，降级（见 §7）

- **站得住（不依赖率门）**：σ=0 每驱动 extinction；同质放电组织 + 慢空间噪声**稳健地**自发出离散自终止事件（drive≥0.6 各噪声档几乎 3/3）——机制上与 rate 场 §11 的"离散几乎不存在"**相反**。
- **撤回**：之前"`robust_2d=False` ⇒ SNN 复现 rate 场 NULL / 无数据兼容离散区 / 事件太快"。该判定全靠率门，而率门有四处不可靠（§7）→ **当前无法判定**是否有数据兼容区、是否复现 NULL。
- **去向**：异质核（Step 3）更硬的动机是**模板多样性**（本测试未测），非事件率。

**sanity（advisor 要求，已满足，与率门无关）**：drive 0.65/0.70 的离散事件计数仅对应 2.4–8/s、frac_time_on≈0.12，远非 27Hz Hopf 振荡的 ~27/s + fton→0.5 → 是噪声触发的稀疏离散事件，不是被计数的 Hopf 周期。

## §7 用户审阅降级（2026-06-06，5 条全核实属实）

1. **探测器标定未完整执行合同**：runner 三处 `calibrate_detector` 都只传 σ=0 地板，漏了合同（`sef_hfo_events.py` §10.2）要求的**亚阈噪声 σ_ref worst-case**。→ 会改变事件切分/率/"太频繁"强度。**结论降级为"当前实现下的扫描"，不能称"完全复用 rate 场合同"。** 已在 `scripts/sef_hfo_snn_noise_scan_runner.py::run_grid` 补 `sig_ref_frac`/`n_ref_seeds`（待 verdict-grade 重跑生效；σ_ref 须验证跨驱动一致亚阈）。
2. **每条仿真仅 1.5 秒，排除低频太短**：[0.01,1]/s 低端要百秒量级才看得清；1.5s 里 1 事件=0.67/s、2 个=1.33/s，量化太粗，唯一 accepted 格恰卡 1.0/s 且 2/3 seed。→ 只能说"已看到的离散区**大多太快**"，**不能**说"已严格排除稀疏低频区"（低噪"extinction"格可能藏着 1.5s 看不见的稀有事件）。
3. **tau 外推过头**：tau 只扫 drive=0.6 / frac=0.05,0.10，没扫边界 frac=0.02、没扫其他驱动低噪边缘。→ "**高率格**的快不由 tau 造成"成立；"整张面 tau-鲁棒"**不成立**（近阈格最可能 tau 敏感，恰恰没测）。
4. **生成引擎+数据在 gitignored 目录**：runner/data/figure 全被 `results/*` 忽略，且整个 SNN 引擎未跟踪。→ 科学结论不变但可复现性差。**已把 runner 移到 `scripts/sef_hfo_snn_noise_scan_runner.py`（tracked）**；SNN 引擎本身（coworker scaffold）是否纳入跟踪 = 用户决定（外部代码）。
5. **"需要病理热点"只是下一步，非已证明；且"每秒数次=明显快于真实间期 HFO"这个前提有问题**：我们 Topic-2 检测的是**穿出来的群体事件**，很多 SOZ 内成核没穿出/没被检到，所以真实组织的**成核率 ≫ 检测到的群体事件率**（数据 0.06–0.3/s 是后者）；模拟数的是**所有**相干成核。两个分母不同 → "2–8/s 太快"**不成立**。要公平比，须给模拟事件套一个"穿得出去/会被当群体事件检到"的可检测性过滤再比率。

**verdict-grade 重跑需要的（按这 5 条）**：(a) 加 σ_ref 亚阈地板（已落代码，须验亚阈一致）；(b) 低噪/低频格用长 T（~30–100s）；(c) 给事件套**可检测性过滤**（空间外延/跨"通道"招募阈值）再比数据率，或改比"会穿出去的大事件"率；(d) tau 扫到边界 frac=0.02 + 其他驱动；(e) 若要支撑异质核，真正该测的是**模板多样性**（成核位点/传播模板数 vs ~2），本测试未测。
