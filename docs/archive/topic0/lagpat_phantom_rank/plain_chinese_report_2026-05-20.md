## 1. 问题到底是什么

### 1.1 数据本来该长什么样

每个 HFO 群体事件发生时，我们要记录"哪些通道在响、它们激活的先后顺序是什么"。

- "哪些通道在响"——一个 True / False 矩阵
- "激活先后顺序"——给每个通道排个名次：1, 2, 3, ... 越小越早

理想情况是：不参与的通道**不该有名次**（应该是 NaN / 空缺）。

### 1.2 21 年的代码犯了什么错

老师范当年的 `hfo_net.py` 生成名次的时候，对**所有通道**都排了名次——不参与的通道也被强行塞了个 1 到 n 之间的整数名次。这个名次来源于该时间窗谱图的噪声中心，**不是真实事件信息**。

**chengshuai 这个 subject 实测**：
- 8 个通道、3513 个事件
- 9064 个"该是空缺"的格子，**全部被塞了 0–7 的整数名次**
- 而且不是均匀塞的，是 U 形：名次 0 占 17.8%、名次 7 占 19.1%、中间每档 ~10%
- 这意味着**两端的名次特别容易是假的**

### 1.3 当前代码为什么没拦下来

我们现在的代码里有四处用 KMeans 把事件分两类（"前向传播" vs "反向传播"），每处都写了一句"把缺失值替换成 0"——但因为老代码给的是有限整数，从来没真的缺失过，这句话**等于没写**。假名次直接喂给 KMeans。

## 2. 我做了什么验证

### 2.1 测试方法

我写了一段代码做这个：
- **原版**：用现在代码里的做法（带假名次）跑 KMeans 分类
- **修过版**：把不参与的格子真的当成空缺，只在参与的通道之间重新排名，然后用中点（0.5）填空，再跑 KMeans
- **比较**：原版分出来的两类，跟修过版分出来的两类，**事件归属是不是一致**

衡量"一致程度"用一个 0–1 的指标：1 = 完全一致；0 = 完全无关。

### 2.2 还做了一个对照（避免自欺欺人）

光比"原版 vs 修过版"不够，因为 KMeans 随机数种子不同也会有点差异。所以我额外测了：**同样原版数据，换 5 个随机数种子跑 5 遍，5 遍之间的一致程度**——这是"噪声基线"。

如果"原版 vs 修过版"的差距还不如"原版换种子"的差距大，那 bug 影响就是噪声级别（cosmetic，化妆品级），可以不管。

### 2.3 先在最坏的 subject 上试

**chengshuai**（之前已知最严重的 case，通道少、参与模式高度结构化）：

| | 数值 |
|---|---|
| 原版换种子的一致程度 | **0.95**（很稳定，5 遍几乎一模一样） |
| 原版 vs 修过版的一致程度 | **0.001**（几乎完全无关） |

也就是说——**原版分类是稳定的，修过版分类也是稳定的，但两者分出的两类几乎完全不重合**。这是最糟的情况：bug 没让结果变随机，bug 制造了一组"稳定但错误"的分类。

### 2.4 然后跑了全部 40 个 subject

| | 全部 40 个 | 主线 35 个（k=2）| 边缘 5 个（k>2）|
|---|---|---|---|
| 一致程度落差中位数 | **-0.60** | -0.61 | -0.25 |
| 差距比噪声基线还大的人数 | **40/40** | 35/35 | 5/5 |
| 差距 > 0.50 的人数 | 27/40 | 26/35 | 1/5 |
| 重新算的"最佳类别数"变了几个 | 4 个 | **1 个**（编号 916：从 2 类 → 4 类）| 3 个 |

**没有一个 subject 通过 cosmetic 阈值**。

## 3. 这意味着什么

### 3.1 不是"加了噪声"，是"分错了"

如果 bug 只是让分类变随机，那原版换种子也会乱跳。但**原版换种子非常稳定（0.997）**。说明 bug 制造了一组**自洽的、可重现的、但和真实通道激活顺序无关**的分类——这组分类是被"哪些通道参与"的几何模式驱动的，不是被"参与通道之间谁先响"驱动的。

打个比方：你要根据食客**进餐厅的先后顺序**给食客分两桌，但 bug 让你根据**食客的身高排名**分了。**身高分类也很稳定**，但不是你想问的问题。

### 3.2 跟之前已知的 PR-4 panel d 问题是同性质的

我们 4 月份已经发现"簇内 86% 的排序一致性来自固定的通道排名（hub identity）"——意思是同一类事件里，通道总是按差不多的顺序响，但这个顺序很大程度上反映的是"哪几个通道天然位置更靠前"，不是"这个事件本身的传播方向"。

**本次审计的发现是这个问题的升级版**：连"事件属于哪一类"本身都被同样的结构性偏置主导。

### 3.3 "两类"这个结构本身还成立

好消息：35 个主线 subject 里 **34 个**用修过版重新算最佳类别数，仍然是 2 类。所以"PR-2 stable_k=2 = 间期刻板时序的主导特征"这个**大结构上的结论站得住**。

坏消息：**具体哪个事件属于哪一类**变了。所有用了"每事件归属哪一类"的下游分析（PR-3 / PR-4 / PR-5 / PR-6 / PR-7 / Topic 4）都得重算。

### 3.4 主线唯一改变最佳类别数的人：epilepsiae_916

916 从 2 类变成 4 类。这个 subject 之前在 Topic 4 attractor Step 1 里也是 GOF 不通过的那一个（拟合不好）。两条线索一对照，说明 916 的传播结构本身比较模糊，原先把它定为 2 类可能就有点勉强。
